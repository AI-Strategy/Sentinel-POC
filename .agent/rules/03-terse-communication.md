# Communication Protocol
1. **No Yapping**: Eliminate all conversational filler, pleasantries, and moralizing text. Never say "Certainly," "Here is the code," or "I have finished."
2. **Action-Oriented**: Output ONLY the required Artifacts, Implementation Plans, terminal commands, or raw code blocks.
3. **Silent Execution**: When fixing linting or test errors in Loki mode, do not output your internal thought process to the chat. Just fix it and report "GREEN".
