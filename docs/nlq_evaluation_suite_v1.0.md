# SENTINEL POC - NLQ EVALUATION SUITE v1.0

**Category**: Systems Architecture & Testing
**Status**: ACTIVE

This evaluation suite validates the Gemini-driven **Text-to-Cypher Engine**. It explicitly tests the LLM's capacity to resolve multi-hop graph traversals, specifically evaluating the path from the abstract `Anomaly` anomaly down to the atomic `SourceDocument` provenance records.

## Evaluation Matrix

The following test questions probe different complexities of graph traversal and Cypher generation.

### Test 1: Direct Evidence Traversal
* **Objective**: Validate the engine can resolve the `[:FLAGS]` or `[:EVIDENCE_FROM]` relationship.
* **NLQ Prompt**: "What are the source file names and record indices for all HIGH severity phantom line flags?"
* **Target Construct**: Matches `Anomaly` nodes where `severity = 'HIGH'` and `type = 'PHANTOM_LINE'`, then traverses to `SourceDocument`.

### Test 2: Multi-Hop Traversal (Flag -> Line & Source)
* **Objective**: Validate the engine can branch queries across both `[:FLAGS]` and other relationships simultaneously.
* **NLQ Prompt**: "List the SKUs and the exact line hints in the source documents for all quantity mismatch flags."

### Test 3: Numerical Filtering & Aggregation
* **Objective**: Validate the engine can apply mathematical logic to node properties before traversing to evidence nodes.
* **NLQ Prompt**: "How many source document records are associated with price variance flags that have a financial impact greater than 100 dollars?"

### Test 4: Origin Filtering
* **Objective**: Validate the engine can filter by a specific entity ID and map the full evidence chain for that specific transaction.
* **NLQ Prompt**: "For invoice INV-2024-0095, show the ghost type, the narrative, and the specific field names from the source documents used as evidence."

## Technical Appendix

### Evaluation Script
The diagnostic script is located at `backend/scripts/test_nlq.py`. It executes the matrix against the `TextToCypherEngine` and outputs the generated Cypher for forensic review.

### Usage

```bash
python -m backend.scripts.test_nlq
```
Outputs the result set and generated Cypher to `nlq_test_results.json`.
