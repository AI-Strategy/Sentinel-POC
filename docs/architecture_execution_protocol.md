## 2. Environment Initialization

Execute the following commands from the root directory to establish the forensic environment.

### A. Substrate Orchestration
Spin up the Postgres vault and Neo4j Graph substrate with GDS enabled.
```bash
docker compose up -d
```

### B. Python Forensic Environment
```bash
cd backend
python -m venv venv
# Windows
.\venv\Scripts\activate
# Unix
source venv/bin/activate

pip install -r requirements.txt
```

## 3. Forensic Execution Workflow

### A. Batch Processing (Forensic Ingestion)
Process all procurement datasets through the Gemini Cognitive Loop.
```bash
# From the root directory
python -m backend.app.main --batch-dir ./data/dirty --reports ./output/reports
```

### B. Grading & Evaluation
Score the generated reports against the ground truth using the automated harness.
```bash
python backend/scripts/grader.py --datasets ./data/dirty --reports ./output/reports --out ./output/results
```

### C. Live API Deployment
Launch the Sentinel API server for direct enterprise integration.
```bash
# From the backend directory
fastapi dev app/main.py
```
